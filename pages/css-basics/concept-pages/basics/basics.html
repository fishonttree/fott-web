CSS content can be done in a separate file, in which we give the head element a link tag, giving the rel attribute (relation) the value "stylesheet" and the href attribute for where the CSS ile is located relative to the HTML file.

Or we can do it on the same file, stuffing all of the properties in a style tag within the head tag.

One's useful for setting common style properties between multiple pages, and one is for unique style properties of particular pages.

The third one, which isn't very useful, is typing the CSS attribute right on the HTML line (inline). It can get messy very quickly.

CSS syntax runs by selectors and semi-colon separated list of declarations - each of them being a property:value pair.

Selectors

The "*" (universal) selector tells us the property applied to all elements.

The type selector select specific elements to affect.

Note that the <div> is a basic HTML element being an empty container. Of course, we want elements that actually do something - but we can run with this one in the meantime.

The class selector select elements by the class attribute given by you. Picking a element in a class can be done with a period ("."). Multiple classes can be assigned to an element. Just remember to select all of them to actually pick the element.

The ID selector works like the class selector, but an element can only have one ID. Picking an element of an ID can be done with a "#".

Class and ID selectors can be chained with each and one another (note that chaining doesn't work on more than one type selector). They can also be grouped together should they share similar style declarations.

Note that the chaining syntax is connected, while the grouping syntax is with line break.

Basics CSS Properties

We can give elements color and background-color... The transparent keyword exists, by the way. No idea what it's for.

The font-family property points out the single value / comma-separated list of values. Each one is either a "font family name" (e.g "Times New Roman" - the quotes is for containing the spaces) or a "generic family name" (e.g sans-serif - generic family names don't use quotes).

It's generally good practice to have a list of fonts, by order of how much we want it up and running, because in the event that the font is not supported, we're gonna need fallback options.

The font-size property will give its size in px (pixels). The number and the unit should not have whitespaces between them.

You can notice the bigger text, right?

The font-weight gives how bold the text will be. It can either be a number or a keyword (e.g bold).

Typically, then number is by hundred increment, between 100 and 900 (within 1 and 1000).

The text can also be given a text-align, through the use of keywords.

Images

Images can be adjusted for height and width, through the keyword auto for it to automatically scale with the other dimension (in px). This is, if you don't want to make it disproportionate for the funni.

Images can be selected in CSS like a class, but without the period

It's generally good practice to include the height and width, so that the image doesn't load slower and also prevents it from weirdly shifting the display.
